# Dashboard Roadmap - Telegram Signal Parser

План реализации веб-интерфейса управления проектом.

## 1. База данных и Модели
- [ ] Добавить таблицу `Settings` в `database/models.py` (key-value хранилище).
- [ ] В `database/connection.py` добавить методы:
    - `get_all_settings()`
    - `update_setting(key, value)`
    - `initialize_defaults()` — первичная инициализация из .env или дефолтов.

## 2. Конфигурация
- [ ] Обновить `config/settings.py`:
    - Добавить метод загрузки из БД.
    - Обеспечить реактивность (обновление параметров при изменении в БД).

## 3. Backend (FastAPI)
- [ ] Создать директорию `telegram_signal_parser/web/`.
- [ ] Реализовать `app.py`:
    - Эндпоинты для отображения страницы.
    - Эндпоинты для сохранения настроек (`hx-post`).
    - Эндпоинты для управления парсером (Start/Stop).
- [ ] Реализовать `ParserManager` для управления фоновым процессом Telethon.

## 4. Frontend (HTML + Tailwind + HTMX)
- [ ] Создать `templates/index.html`:
    - Использование Tailwind CSS (CDN) для стилей.
    - HTMX для динамических обновлений без перезагрузки.
    - Индикаторы статуса (Работает/Остановлен).
    - Форма настроек с группами (API, База, Фильтры).

## 5. Интеграция и Запуск
- [x] Обновить `run.sh` для запуска веб-сервера на порту 7001.
- [ ] Убедиться, что при нажатии "Apply" настройки вступают в силу.

## 6. Проверка и Тесты
- [ ] Проверить сохранение настроек.
- [ ] Проверить запуск и остановку парсера из веб-интерфейса.
- [ ] Проверить адаптивность дизайна.

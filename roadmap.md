# Telegram Signal → MT5 Auto Trading System
## Дорожная карта реализации

---

## 📋 Переформулировка требований

### Общая архитектура
Система состоит из **двух полностью изолированных компонентов**:
1. **Python-сервис** — читает сигналы из Telegram, парсит и сохраняет в SQLite3
2. **MQL5 EA** — читает сигналы из БД через файл, исполняет сделки

### Ключевые требования
- Связь между компонентами **только через локальный файл** (экспорт из SQLite)
- EA **без DLL и WebRequest**
- Риск-менеджмент: максимум 3% потерь в день
- Только **один активный сигнал** в любой момент
- Каждый сигнал = **2 ордера** (TP1 и TP2)

# 📦 ЧАСТЬ 1: Python Telegram Parser

## 1.1 Рекомендуемый технологический стек

### Основные библиотеки

| Компонент | Библиотека | Версия | Назначение |
|-----------|------------|--------|------------|
| Telegram API | **Telethon** | 1.36+ | MTProto клиент для чтения каналов |
| База данных | **sqlite3** | встроенная | Хранение сигналов |
| Валидация | **Pydantic** | 2.x | Валидация и сериализация данных |
| Конфигурация | **python-dotenv** | 1.0+ | Управление секретами |
| Логирование | **loguru** | 0.7+ | Структурированные логи |
| Асинхронность | **asyncio** | встроенная | Асинхронная обработка |

### Почему Telethon?
- ✅ Стабильная библиотека с 2016 года
- ✅ Полная поддержка MTProto 2.0
- ✅ Чтение каналов/групп без бота (userbot)
- ✅ Активная поддержка и документация
- ✅ Простой API для чтения сообщений

### Альтернатива: Pyrogram
- Более современный API
- Лучше типизация
- Но: менее стабильные обновления в 2025

## 1.2 Структура базы данных SQLite3

### Таблица: `signals`

| Поле | Тип | Описание |
|------|-----|----------|
| id | INTEGER PK | Первичный ключ |
| telegram_message_id | INTEGER | ID сообщения в Telegram |
| telegram_channel_id | INTEGER | ID канала |
| symbol | TEXT | Торговый символ (XAUUSD) |
| direction | TEXT | BUY или SELL |
| entry_min | REAL | Минимальная цена входа |
| entry_max | REAL | Максимальная цена входа |
| stop_loss | REAL | Уровень Stop Loss |
| take_profit_1 | REAL | Первый Take Profit |
| take_profit_2 | REAL | Второй Take Profit |
| take_profit_3 | REAL | Третий Take Profit (опционально) |
| status | TEXT | NEW, SENT_TO_EA, PROCESSED, EXPIRED, INVALID, ERROR |
| raw_message | TEXT | Оригинальный текст сообщения |
| created_at | TIMESTAMP | Время создания записи |
| updated_at | TIMESTAMP | Время обновления |
| processed_at | TIMESTAMP | Время обработки EA |
| parse_error | TEXT | Текст ошибки парсинга |

### Дополнительные таблицы
- `signal_export_log` — лог экспорта сигналов для EA
- `channels` — настройки отслеживаемых каналов
- `app_settings` — настройки приложения

> **Полная SQL-схема и примеры кода:** см. [code_examples.md](code_examples.md)

## 2.2 Логика работы EA

```
OnTick()
    │
    ├── 1. Проверка риск-менеджмента
    │   └── Дневной убыток >= 3%? → СТОП торговли
    │
    ├── 2. Чтение файла сигналов
    │   └── Новый сигнал? → Продолжить
    │
    ├── 3. Обработка существующих позиций
    │   └── Есть открытые? → Закрыть все (новый сигнал)
    │
    ├── 4. Проверка условий входа
    │   ├── Цена в диапазоне? → Вход по рынку
    │   └── Цена за диапазоном?
    │       ├── Отклонение <= 30 pips? → Вход по рынку
    │       └── Отклонение > 30 pips? → Игнорировать
    │
    ├── 5. Открытие 2 ордеров
    │   ├── Order 1: TP1, SL
    │   └── Order 2: TP2, SL
    │
    └── 6. Мониторинг позиций
        └── TP1 достигнут? → Перенести SL Order 2 на безубыток
```

# 🛡️ ЧАСТЬ 3: Риск-менеджмент

## 3.1 Правила риска

| Правило | Значение | Действие при нарушении |
|---------|----------|------------------------|
| Дневной убыток | ≤ 3% от эквити | Закрыть все, стоп торговли |
| Проскальзывание | ≤ 30 pips | Отклонить ордер |
| SL обязателен | Всегда | Немедленно закрыть сделку |
| Дубликаты | Запрещены | Игнорировать сигнал |

## 3.2 Логика мониторинга эквити

1. На начало дня сохраняем `dailyStartEquity`
2. Каждый тик проверяем текущую эквити
3. Если убыток >= 3%:
   - Закрыть все позиции
   - Установить флаг `tradingDisabled = true`
   - Ждать следующего торгового дня
